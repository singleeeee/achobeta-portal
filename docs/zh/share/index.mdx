# 一文读懂莞工 AchoBeta 实验室

# 🚩愿景和目标

## 1. 什么是 AchoBeta 

:::tip{title=团队}
AchoBeta 是一个集学习、实践、比赛、多种类活动、就业于一体学生团队（老师+学生+校友+企业）
:::

:::tip{title=精神}
AchoBeta 是一种创新文化精神
:::

:::tip{title=纽带}
AchoBeta 是联结学生、学校、社会的一条纽带
:::

## 2.  AchoBeta 能做什么

:::info{title=为学生}
提供路径清晰的成长方法论、配套资源、多类型活动、提升主动性
:::

:::info{title=为校友}
提供校园试点创业机会、维系校友网络、实现校友情怀落地
:::

:::info{title=为企业}
高效选育优质人才、提供知识应用平台
:::

:::info{title=为学校}
提升学校学风建设、培育应用型人才、提高就业率
:::

## 3. AchoBeta 要做什么

### 3.1 阶段一：轻运营方案—知识库+知识星球+技术团队
1. 阶段状态：目前在做
2. 知识库：技能水平提升SOP、实操项目SOP
3. 知识星球：存放资源、沟通需求
4. 技术团队：参与专业竞赛
5. 方案分析：运营成本相对最小，也有一定影响力

:::tip{title=优点}
团队短小精悍、沟通成本小
:::

:::danger{title=缺点}
团队规模较小、影响力受限
:::
### 3.2 阶段二：中度运营方案—知识库+知识星球+技术团队+项目团队
1. 阶段状态：目前在做，但是效果不及预期；仍在探索阶段
2. 知识库：技能水平提升SOP、实操项目SOP知识库
3. 知识星球：存放资源、沟通需求
4. 技术团队：可以理解AB选拔具备产品/技术领导力的人才、作为项目负责人或者核心骨干参与不同项目
5. 项目团队：项目负责人围绕核心项目拓展一个相对独立的项目组
6. 方案分析：在理想模型中，这种模式的可拓展性是最强的，投入成本适中；实操下来会有人员诉求不一致、职业规划时间冲突等诸多问题的影响，需要克服众多困难（立项合理性、执行SOP、绩效管理、意外管理等）

:::tip{title=优点}
团队人多力量大、可拓展性强
:::

:::danger{title=缺点}
人员筛选成本高、管理成本高、对项目负责人和组员的要求高、项目驱动性弱
:::


### 3.3 阶段三：重度运营方案—知识库+知识星球+小程序交易平台+内部团队

1. 阶段状态：还未正式启动
2. 知识库：技能水平提升SOP、实操项目SOP
3. 知识星球：存放资源、沟通需求
4. 小程序交易平台：

- 莞工知识产权交易平台
> 允许不同项目负责人将自身的项目挂在网上卖掉，获取利润
待解决问题：设计一套合理的分销制度；获客引流（比如会有想买AI应用创业的小老板，愿意花个十来万几十万，怎么找到这些老板，怎么确定交付物，怎么签订交易合同……）
- 莞工知识众包平台
> 针对有技能又想要及时变现的同学（早期可以围绕几类人群引流，一个是开发外包、一个是家教老师）
待解决问题：撮合供需方；获客引流；商业模式设计（C2C会有绕过平台成交风险；B2C会有比较高的运营成本）
5. 内部团队：
-  通过商业化的方式运营团队
-  参与交易平台搭建的成员可以直接获得报酬
-  项目制的成员可以通过平台快速兑现项目价值
6. 方案分析：通过商业利益关系绑定成员，可以实现综合收益的最大化，实操下来可能会存在理想与现实脱节的风险，属于正式的创业项目


:::tip{title=优点}
综合考虑多方利益，设计一套商业模式，让大家各得其所
:::

:::danger{title=缺点}
投入成本高、回报周期长；ROI不明确；方案设计难度高
:::

## 4. 团队文化
### 4.1 工作理念
#### 4.1.1 充分领导力

> 希望每一位AchoBeta的大家庭的成员都铭记这句话，让自己成为一名领导者。

:::danger{title=ohn Quincy Adams}
  If your actions inspire others to dream more, learn more, do more and become more, you are a leader. 

  如果你的行为激励他人梦想更多，学会更多，实践更多，成就更多，那你毋庸置疑是一位领袖。
  ——约翰-昆西-亚当斯，美国第6位总统
:::

#### 4.1.2 目标导向
> 聚是一团火，散是满天星（首先是要保证一团火可以烧起来，也就是要做好一个个的项目）

#### 4.1.3 发挥潜能
> 野蛮生长，协同创新
#### 4.1.4 友好协同
> 用冰冷的技术，赋予生活无限的温柔（学习的压力已经够大了 ，实践、工作、生活中请务必温柔耐心一些）
### 4.2 工作方式
优秀者同盟，高度负责任，高度自驱动，以项目成果和用户体验为行动关键导向。
开诚布公地讨论交流问题，积极主动地给合作伙伴反馈（windows+shift+s+鼠标划区域=局部截图，mac类似，多截图多反馈是良好的工作习惯，无论是学习生活还是工作）。
发现问题、分析问题、解决问题、问题复盘、继续向前，从不抱怨。
### 4.3 经验分享
#### 4.3.1 主人翁意识
大家进来一个团队，不是说等别人来慢慢培养自己的（这一点大家要和学校组织区别开来），也不是说要等团队给自己分配任务，一切要需要自己和身边小伙伴的主动。
做得越多，收获越多；能力越大，责任越大。
（项目负责人需要对整个项目的绩效评估负责，具备leadership和统筹能力的担任项目责任人，与所在组别无关，与年龄资历角色无关。）
大家是队友、同辈，团队找的partners，不是followers。
项目为导向，一切以项目成功为最终导向。自愿原则，高度负责。
（从项目发展来说，如果大家觉得项目不行，项目负责人可以选择向外扩招人员参与项目，或者冷却（中止）项目，团队没有硬性规定；加入了项目的不同阶段就有不同阶段的工作，请大家务必全力助力该项目的发展）
#### 4.3.2 尽早提升能力
特别是第一个学期最后几周稍微有点空余时间的时候，一定要抓紧提升自己的实际项目能力
#### 4.3.3 做好时间管理
每学年下学期是比赛的高频时期，但是看到比赛发出通知才开始准备是完全不够时间的，比赛要想取得好成绩，关键在于资源（专利门槛、软件著作、专业师资、成熟的商业模式），所以建议项目提前进行运营，获取更多数据，项目才能走得更远。
#### 4.3.4 如何和老师合作
在有老师的微信群

> #汇报工作进度、关键问题指导会议

在没有老师的小群

> #日常高频交流、项目催进度
PS：不要想着让老师给我们直接出idea，正确请教老师的方式是自己已经进行了大量前期调查，提供2~4种解决方案，请求老师的点评建议
